# FSI demo-data realism rules

> Status: **🔶 Draft — to be canonized during the FSI KYC pilot.**
> This file is the seed; the KYC pilot will harden every rule against
> a real demo run with FSI SMEs in the room.

## Identifiers

| Type | Rule | Example |
|------|------|---------|
| **SSN (US)** | Use ITIN or "documentation" range only — never real SSN format | `900-XX-XXXX`, `999-XX-XXXX` |
| **EIN (US)** | Always start with prefixes that aren't issued: `00`, `07`, `08`, `09`, `17`, `18`, `19`, `28`, `29`, `49`, `69`, `70`, `78`, `79`, `89` | `00-1234567` |
| **IBAN (EU)** | Use the test IBANs published by each country bank | `DE89 3704 0044 0532 0130 00` (test) |
| **Sort code (UK)** | Use `00-00-XX` reserved range | `00-00-99` |
| **SWIFT/BIC** | Use codes ending in `XXX` (test branch) | `DEUTDEFFXXX` |
| **Customer ID** | Free-format internal — prefix with `DEMO-` so it's obvious | `DEMO-CUST-00042` |
| **Account number** | 8-12 digit synthetic, prefix with `9` (rarely real) | `987654321` |

## Names

| Field | Rule |
|-------|------|
| **Person names** | Faker `en_US`, `en_GB`, `de_DE`, etc. matching customer's geography. Never use names of real public figures. |
| **Company names** | Two-token "shifted" pattern: `<Adjective> <Industry-noun>` — e.g. `Northbank Holdings`, `Pacific Mutual`, `Cardinal Capital`. Never reuse a real bank name verbatim. |
| **Branch names** | Real city + fictional street: `Manhattan – Hawthorne Street Branch` |

## Sanctions / watchlist

- **Always include a sanctioned-entity case** in the demo data, but the entity
  must be **fictional** (e.g. `Crimson Holdings Ltd` registered to a fictional
  Caribbean address). The entity should match the *shape* of a real OFAC entry
  (incorporation date, named officers, address structure) without using any
  real OFAC entry verbatim.
- The agent's hit on this entity is the dramatic moment in the FSI experience
  cinematic — preserve it as a named golden case.

## Transactions

- **Amount distribution**: log-normal, mean ~$1,200, p99 ~$50,000.
- **Velocity**: normal customers do 5-15 txns/month; high-velocity flag fires
  at ≥40/month.
- **Geographies**: 80% home country, 15% common cross-border (UK→EU, US→Canada),
  5% high-risk corridors (named fictional territories).
- **Suspicious patterns** seeded for the demo: structuring (10× $9,800), round
  numbers + offshore destination, dormant-then-active accounts.

## Credit data

- **FICO scores**: mean 695, std 60, clip [300, 850]. Plausible bimodal at
  580 (sub-prime cluster) and 760 (prime cluster).
- **DTI ratios**: mean 28%, std 12%, clip [0%, 80%].
- **Income**: log-normal, geography-adjusted, with a long tail.

## Voice / document samples (for KYC, FNOL)

- **ID document images**: use the templates published by Microsoft Document
  Intelligence sample gallery (passport, driver's license). Replace name,
  DOB, photo with synthetic. **Never** reuse a real ID image, even partially.
- **Selfie / biometric photos**: use AI-generated faces (this-person-does-not-exist
  pattern) — never real photos.
- **Voice samples** (FNOL): TTS-generated from synthetic scripts. If using
  realistic voice acting, get explicit talent release. Never use real claim
  call recordings.

## Document corpora (for Foundry IQ Knowledge Bases)

- **Policies / regulations**: prefer **publicly published** primary sources
  (Federal Register, EUR-Lex, FCA Handbook, FFIEC BSA/AML manual). These are
  always safe to ship. Do NOT use a customer's internal policy document even
  if the customer shared it for a workshop — that document stays in their
  tenant.
- **Brand guidelines / product docs**: synthesize a fictional bank's policy
  (e.g. "Northbank KYC Manual v3.2") with realistic structure. The Knowledge
  Agent's citation pattern is what matters; the content can be plausible
  fiction.

## Reset semantics

- `idempotent` — reset wipes Cosmos containers and re-seeds from JSON files.
  Reset must complete in <30s for live demo recovery.
- The reset script is generated by `threadlight-demo-data-factory` and lives
  at `scripts/reset_data.py`. It must be safe to run while the agent is up
  (no foreign-key dependency on transient state).
