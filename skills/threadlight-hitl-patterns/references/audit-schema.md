# Audit trail schema

> Every action gate writes one audit event to Cosmos `case_audit` container.
> This is the canonical schema — handlers MUST conform.

## Container shape

| Property | Value |
|----------|-------|
| Container name | `case_audit` |
| Partition key | `/case_id` |
| TTL | `null` (never expire — audit is permanent) |
| Indexing policy | include `gate`, `actor.upn`, `timestamp`, `linked_rules[]/?` |

## Document shape

```json
{
  "id": "audit-{uuid4}",
  "case_id": "DEMO-CUST-00007",
  "gate": "approve",
  "decision": "approved",
  "actor": {
    "upn": "analyst@northbank-demo.com",
    "displayName": "Jane Analyst",
    "objectId": "00000000-0000-0000-0000-000000000000"
  },
  "timestamp": "2026-05-10T08:47:23Z",
  "linked_rules": ["BR-001", "BR-007"],
  "agent_proposal": {
    "decision": "approved",
    "rationale": "Customer cleared all KYC checks; no sanctions hit; PEP score 0.12 (below threshold)",
    "tool_calls": [
      {"tool": "sanctions_check", "result": "no_match"},
      {"tool": "pep_score", "result": 0.12}
    ]
  },
  "human_edits": null,
  "rationale": null,
  "sla": {
    "deadline": "2026-05-10T12:47:23Z",
    "status": "met"
  }
}
```

## Per-gate field requirements

| Field | approve | edit-and-approve | reject | escalate | signoff | audit-view | request-info |
|-------|---------|------------------|--------|----------|---------|------------|--------------|
| `decision` | ✅ | ✅ | `declined` | `escalated_to: <role>` | `acknowledged` | `viewed` | `requested` |
| `agent_proposal` | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| `human_edits` | n/a | ✅ (diff) | n/a | n/a | n/a | n/a | n/a |
| `rationale` | optional | recommended | ✅ required | ✅ required | optional | n/a | ✅ required |
| `sla` | ✅ | ✅ | ✅ | ✅ | ✅ | n/a | ✅ |
| `escalation_target` | n/a | n/a | n/a | ✅ | n/a | n/a | n/a |
| `request_template` | n/a | n/a | n/a | n/a | n/a | n/a | ✅ |

## Reading the audit

The audit container is queried by:

- **Workspace UI** (audit viewer drawer / panel)
- **Continuous evals** — `foundry-evals` ACA job pulls these to compute
  KPIs (approval rate, escalation rate, SLA-met rate, etc.)
- **Compliance team** — direct Cosmos queries for regulator reports
- **Replay tooling** — to reconstruct what happened on a case

## Privacy

The `actor.upn` field IS a real UPN in production but should be a
synthetic UPN (`analyst@northbank-demo.com`) in demos. Generated by
`threadlight-demo-data-factory` via the universal "no real PII" rule.
