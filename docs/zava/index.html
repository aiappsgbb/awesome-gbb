How it works

Name the company. The coding agent builds the workspace.

You give a company name. The coding agent (GitHub Copilot CLI, VS Code, Claude Code) researches the org, forks a 37-domain agentic substrate with their brand and executives, and deploys a control-plane dashboard to Azure Container Apps — with persona-driven orchestration, SSE streams, and telemetry.

✗ The old way
Manual research. A slide with org charts. Mock screenshots. The customer sees a generic demo with placeholder names and never feels it’s theirs.
✓ With Zava skills
One pipeline. You say: “Profile Contoso Bank, fork the substrate, deploy to Azure.” The coding agent produces a branded workspace with their CFO, their compliance rules, their industry’s entity kinds — running live at a URL they can bookmark.

The skills pipeline

Three skills. Company name to deployed workspace.

Each skill is a composable building block — Markdown instructions that any coding agent reads and follows. You name a company. The agent discovers the matching skills, chains them, and hands you a branded control-plane running on Azure.

research-company
Entry skill 15–40 min

Profile the target company against its public web footprint — Wikipedia, annual reports, registry data — and emit a thin org-brief.yaml with named ELT, subsidiaries, strategic themes, and stack overrides.

InputCompany name or domain
→
Outputbriefs/<slug>-org-brief.yaml
> "Use research-company to profile Contoso Bank."
Every claim tagged high/medium/low/inferred with source refs. SKILL.md →
compose-org
Fork substrate 30–60 min

Fork the zava-control-plane substrate into a customer-branded digital clone — rebrand tokens, repack data fabric, swap entity kinds, regenerate personas and functions, scaffold 25–35 domain stubs.

Inputorg-brief.yaml + industry primer
→
Outputzava-control-plane-<slug>/ (standalone repo)
> "Use compose-org to fork the substrate for Contoso Bank using the banking primer."
Ten phases. Pauses for operator approval at each destructive step. SKILL.md →
zava-workspace-deploy
azd deploy ~5 min

Build the React SPA, package with FastAPI, deploy to Azure Container Apps with OpenTelemetry → App Insights. Ships 170+ routes, SSE fleet streams, and a polished dashboard — all from one azd deploy.

InputBranded fork + Azure subscription
→
OutputLive ACA workspace + App Insights telemetry
> "Use zava-workspace-deploy to ship this to Azure."
Single container: FastAPI serves API + React SPA + SSE + OTel. SKILL.md →

Enterprise story

Zava + Threadlight = the full operating system.

Threadlight designs and deploys individual business-process agents. Zava wraps them in a multi-domain control plane with fleet orchestration, persona decisions, and a polished dashboard. Use Threadlight for the “one process” pitch; Zava for the “enterprise operating system” pitch. Together, they tell the complete story.

  Morning

Research + Compose

research-company profiles the customer. compose-org forks the branded substrate. The customer sees their own org structure.

  Afternoon

Design + Deploy Agents

threadlight-design specs 2–3 hero processes. threadlight-deploy ships Foundry hosted agents. Process-level personas handle individual cases.

  End of day

Deploy Dashboard

zava-workspace-deploy ships the branded control plane. Executive personas orchestrate the fleet. The customer bookmarks the URL.
