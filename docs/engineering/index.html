How awesome-gbb stays green

Curation discipline

AGENTS.md §§ 2, 5, 7, 12.1

A skill earns its place in the catalog only if it’s reusable, honest, and small enough to load. These rules are mechanical — every PR runs through them, human or coding-agent authored.

Every skill is a contract

Code samples in SKILL.md are not suggestions — they are the exact bytes a consumer will copy. If they don’t run, the skill is broken. Quality is the only differentiator.

Single source of truth for code

Non-trivial snippets live in references/<lang>/ and SKILL.md cross-links them with a one-line MUST. A validator compiles every reference file and rejects inline duplication — excerpts drift, then ship wrong.

Description ≤ 1024 characters

The runtime caps skill descriptions at 1024 chars (some loaders at 512). Every USE FOR / DO NOT USE FOR trigger phrase must fit. We count bytes on every PR.

SemVer discipline

MAJOR for breaking renames or removed sections. MINOR for new capabilities. PATCH for typo fixes and pin refreshes. The CI gate rejects non-PATCH bumps for metadata-only changes — history stays honest.

Reference data is canon — do NOT normalize

Industry shorthand (EIN prefixes, sort codes, NANPA ranges) cites real standards. A scrub agent once “fixed” single-digit EIN prefixes to double-digit; the revert took a full turn. Mass-edit invariants now block it at PR gate.

Agnostic by design

No customer names, no PoC names, no real GUIDs or ARM IDs. Skills are reusable building blocks — what stays in the catalog must read sensibly to a stranger who has never seen the engagement it was first authored for.

azd is the default deploy model

Every infra-bearing skill ships azure.yaml + Bicep, not hand-rolled az CLI scripts. The azd-patterns skill is the single source of truth for module shapes, hooks, and env conventions.

Six CI gates

.github/workflows/

skill-validation.yml

Frontmatter parses, description ≤ 1024, valid SemVer, no forbidden strings, pin files conform to schema v2, plugin.json + marketplace are version-consistent.

every PR

automation-pr-gate.yml

Enforces the mass-edit invariants — rejects multi-skill body edits, normalization of reference canon, and non-PATCH bumps for metadata-only changes. Opt-in commit tags make intent explicit.

every PR

pin-validation.yml

Re-runs the pin file’s validation.script on the runner and asserts every expected_output substring. No “trust me, I tested” path.

pin changes

skill-freshness.yml

Detects SHA drift, PyPI bumps, upstream issue closure, link rot, and stale validation. Opens one consolidated issue per skill with an impact: label — auto-tier issues are assigned to @Copilot.

weekly cron

skill-test.yml

Unit tests, catalog lint, the all-pin smoke job, and the E2E Azure matrix — real azd up, real Foundry calls, real model inference in .

every PR · push · cron

auto-merge-copilot.yml

Auto-approves and squash-merges Copilot-authored PRs once every gate is green. Closes the freshness loop end-to-end.

on check-suite success

Four testing tiers

each subsumes the one below

T0   Lint

Frontmatter shape, description length, forbidden-string scan, deprecated-API detection. Runs on every PR.

T1   Pin validation

validation.script executes on the runner; every expected substring must appear in stdout. Runs whenever a pin file changes.

T2   Import smoke

pip install + python -c "from X import Y" for every documented dependency. Catches dead imports the moment upstream bumps.

T3   E2E Azure

Deploys real resources, calls real Azure APIs, exercises real credential chains. The bar: if SKILL.md tells consumers to connect to Azure, CI must prove that connection works.

25 hard-won fixture patterns

AGENTS.md § 9.7

Every CI flake we hit on live Azure became a numbered pattern. Future fixtures inherit them by default — the catalog doesn’t rediscover the same bug twice. A representative slice:

Pattern 4   Marker priming defenses

Render the literal token as _MOKE_RESULT in fixture prose so autoregressive LLM continuations can’t accidentally wrap the real marker in backticks.

Pattern 12   File-write marker, not prose

Final fixture step is a Bash tool call: printf 'SMOKE_RESULT=PASS\n' > /tmp/<skill>-smoke-result. The file is graded byte-exact — prose rendering can’t corrupt it.

Pattern 14   Timeout ≥ p99 × 1.2

Job-level timeout-minutes must exceed observed p99 by 20%, or PASS markers get killed at the wire and CI shows cancelled.

Pattern 17   Show-don’t-assert on CLI cache

az credential cache may or may not be inherited by subshells. Print it for audit — never gate flow on it. azd auth login is the real auth gate.

Pattern 22   Throttle matrix parallelism to 2

Copilot CLI’s own backing model has tight RPM quota. Three parallel copilot -p invocations deterministically hit CAPIError 429.

Pattern 23   Foundry project MI needs Cog roles

Server-side workers (memory consolidation, hosted-agent runtime) run as the project MI — a third identity beyond your UAMI. Grant Cognitive Services OpenAI User explicitly.

Pattern 25   Teardown is best-effort

Skill contract: hard FAIL. Cleanup of side resources: soft-PASS with a NOTE. OIDC TTL bounds discovery; a CI janitor reaps orphans.

All 25 patterns live in AGENTS.md § 9.7 with provenance, anti-patterns, and forensic diagnostic protocols.
