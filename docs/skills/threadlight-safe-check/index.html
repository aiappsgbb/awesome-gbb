threadlight-safe-check

v1.0.3 🧵 Threadlight Pipeline

Three-lifecycle completeness gate for threadlight pilots (design / pre-deploy / post-deploy). Reads SPEC § 11c selectors via `specs/manifest.json` `deployment_manifest` and asserts: every selector maps to a deployed `Microsoft.*` type, every channel reaches, every scheduled job is wired. Post-deploy also runs behavioural checks: deployed images differ from the azuredocs helloworld placeholder, no ACA Job has its last 5 runs all Failed, App Insights exists when SPEC declared it. USE FOR: completeness gate, deploy gate, post-deploy gate, pre-deploy check, manifest drift, orphan modules, partial PoC, missing bot/workspace/aca-job, deployment_manifest, manifest.json, placeholder image, helloworld image, image probe, job execution failed, cron rot, app insights missing, telemetry not flowing, blank appin. DO NOT USE FOR: invocation/runtime tests (foundry-evals), `azd up` orchestration (threadlight-deploy), schema authoring (threadlight-design).
